# 축 1 보강 2: 교육용 출력 프리셋 (Output Presets)

> **전제**: 지문 → 구조 다이어그램 생성 성공 후, 교사가 결과물을 실제 수업에 투입하는 시점의 UX. 주 사용자 = 고등 영어 교사, 학원 강사, 감독님(수업 도구).
> **이 문서의 목표**: "출력 편하게"를 3가지 교사 시나리오로 분해하고, 각 시나리오에 맞는 프리셋 UI + 결과물 포맷 설계.

---

## 1. 교사 3인의 하루 (시나리오 정의)

### 시나리오 A — 교사 A: "정규 수업 준비" (저녁 9시)

- **상황**: 내일 3교시 모의고사 분석. 지문 5개 중 2개가 학생들 오답률 높음.
- **니즈**: 학습지 PDF로 뽑아서 복사실 맡기기. A4 한 면에 다이어그램 + 지문 + 문제번호.
- **제약**: 10분 내 완성 필요. 컴퓨터실 프린터 기본 설정.
- **출력 결과물**: **인쇄용 PDF (A4, 흑백 호환)**

### 시나리오 B — 강사 B: "1:1 수업 중" (실시간)

- **상황**: 화상 수업 중. 학생이 지문 막혀서 질문.
- **니즈**: 실시간으로 다이어그램 생성 → 화면 공유 → 끝나고 학생에게 링크 전달.
- **제약**: 수업 끊김 없이 30초 이내 "링크 공유" 완료.
- **출력 결과물**: **공유 URL (Lightbox, 카톡 복붙 가능)**

### 시나리오 C — 감독님(콘텐츠 제작): "강의 교재 개발" (주말)

- **상황**: 블로그/PDF 교재/유튜브 썸네일용 이미지 제작.
- **니즈**: 고해상도 이미지, 브랜드 색상 일관성, 여러 지문을 같은 스타일로 연속 생성.
- **제약**: 디자인 툴 없이 한 번에 예쁘게 뽑고 싶음.
- **출력 결과물**: **고해상도 PNG + SVG (편집 가능)**

## 2. 프리셋 3종 정의

각 시나리오에 대응하는 **원클릭 프리셋**. 사용자는 포맷/옵션을 개별 선택하지 않고 **시나리오만 선택**.

### 2.1 📄 학습지용 (Handout)

**대상**: 교사 A
**포맷**: PDF
**특징**:
- A4 세로 방향 (210×297mm)
- 상단 여백 20mm (학생 이름/반 기입)
- 다이어그램 영역: A4의 상단 60% (제목 포함)
- 하단 40%: 원문 지문 (10pt, 줄번호 포함)
- 흑백 인쇄 대응 (색상 + 패턴/선두께 조합)
- 우측 상단: 문제번호 / 출처 (편집 가능)
- 페이지 하단: QR 코드 → 온라인 인터랙티브 버전 링크

**레이아웃 스케치**:
```
┌───────────────────────────────────────┐
│  [2024학년도 수능 34번] 이름:___반:___ │
├───────────────────────────────────────┤
│                                       │
│        [구조 다이어그램 이미지]        │
│                                       │
├───────────────────────────────────────┤
│  1. Traditional economic theory       │
│     assumed that humans make...       │
│  2. However, behavioral economists... │
│  ...                                  │
├───────────────────────────────────────┤
│  [QR] 인터랙티브 버전       제작:교사A  │
└───────────────────────────────────────┘
```

---

### 2.2 🔗 공유 링크 (Share Link)

**대상**: 강사 B (실시간), 학생 (자기주도학습)
**포맷**: Lightbox URL (드로우 자체 기능)
**특징**:
- `https://embed.diagrams.net/?lightbox=1&nav=1&highlight=FF9900#R<pako>`
- 읽기 전용 (편집 막음)
- 레이어별 단계 탐색 가능 (학생이 화살표 키로 reveal)
- URL 길이 ~2000자 → **단축 URL 변환 옵션** (TinyURL 또는 자체 단축기)
- **원클릭 복사** 버튼 (클립보드 API)
- **카톡 공유** 버튼 (모바일은 Web Share API)

**URL 구조**:
```
https://embed.diagrams.net/?
  lightbox=1       ← 뷰어 모드
  &highlight=0000FF ← 강조색 (수업 주제 맞춤)
  &nav=1            ← 페이지 네비게이션 (레이어별 탐색)
  &layers=1         ← 레이어 패널 표시
  &edit=_blank      ← "편집" 버튼 비활성화 (학생 임의편집 방지)
  #R<compressed>    ← pako-deflated + base64 XML
```

---

### 2.3 🎨 고화질 이미지 (Hi-Res Export)

**대상**: 감독님 (콘텐츠 제작)
**포맷**: PNG (2x, 3x) + SVG (벡터)
**특징**:
- **PNG 2x**: 1920×1080 기준 프리셋 (블로그, 유튜브 썸네일)
- **PNG 3x**: 2880×1620 (PDF 교재 삽입용)
- **SVG**: 벡터 편집 가능 (Figma, Illustrator 반입)
- 투명 배경 옵션
- 다크모드 버전 동시 생성 (라이트/다크 2장)
- 워터마크/브랜드 로고 삽입 옵션 (우측 하단)

## 3. UI 디자인

### 3.1 현재 상태 (Before)

채팅창 하단 툴바의 ⬇ 다운로드 버튼 → 모달에서 포맷 4개 선택 → 파일명 입력 → 저장.

**문제점**: 교사가 매번 포맷/해상도를 판단해야 함. 학습지와 블로그용이 같은 "PNG"인데 해상도가 달라야 한다는 것을 사용자가 알아야 함.

### 3.2 개선안 (After)

```
┌─ 다운로드 버튼 (⬇) 클릭 ────────────┐
│                                      │
│   어디에 쓰실 건가요?                 │
│                                      │
│   ┌──────────────────────────────┐   │
│   │ 📄 학습지 (A4 인쇄용 PDF)    │   │
│   │    교사 배포용                │   │
│   └──────────────────────────────┘   │
│                                      │
│   ┌──────────────────────────────┐   │
│   │ 🔗 공유 링크 (학생에게 전송)  │   │
│   │    카톡/밴드/블로그 바로 공유 │   │
│   └──────────────────────────────┘   │
│                                      │
│   ┌──────────────────────────────┐   │
│   │ 🎨 고화질 이미지 (PNG/SVG)    │   │
│   │    교재·블로그·발표자료용      │   │
│   └──────────────────────────────┘   │
│                                      │
│   ▸ 더 많은 옵션 (고급)              │
│                                      │
└──────────────────────────────────────┘
```

**"더 많은 옵션"** 클릭 시 → 현재의 포맷 4개 + 해상도/옵션 세부 제어 (파워유저용).

## 4. 각 프리셋의 내부 처리 플로우

### 4.1 📄 학습지용 PDF

```
[사용자 클릭]
  ↓
1. 다이어그램을 고해상도 PNG로 export (react-drawio postMessage)
  ↓
2. jsPDF 인스턴스 생성 (A4, portrait)
  ↓
3. 헤더 영역 그리기 (제목 placeholder, 이름/반 칸)
  ↓
4. PNG를 페이지 상단 60% 영역에 삽입 (aspect ratio 보존)
  ↓
5. 원문 지문을 하단 40%에 렌더 (word-wrap, 줄번호)
  ↓
6. QR 코드 생성 (공유 URL → 하단 우측)
  ↓
7. PDF Blob 다운로드 트리거
```

**의존성**: `jspdf`, `html2canvas` (또는 PNG 직접 삽입), `qrcode`

### 4.2 🔗 공유 링크

```
[사용자 클릭]
  ↓
1. 현재 diagram XML 획득 (drawioRef.current 또는 context state)
  ↓
2. pako.deflateRaw(xml) → Uint8Array 압축
  ↓
3. btoa(압축 결과) → base64 (URL-safe 치환: +→-, /→_)
  ↓
4. URL 조립: https://embed.diagrams.net/?lightbox=1&nav=1&layers=1#R<base64>
  ↓
5. navigator.clipboard.writeText(url) → 클립보드 복사
  ↓
6. Toast: "복사 완료! 카톡에 붙여넣으세요"
  ↓
(선택) 네이티브 공유 API: navigator.share({ url, title })
```

**의존성**: `pako` (이미 설치됨), 브라우저 내장 Clipboard API

### 4.3 🎨 고화질 이미지

```
[사용자 클릭]
  ↓
1. 해상도 선택 (2x / 3x 기본 옵션)
  ↓
2. react-drawio exportDiagram({ format: 'png', scale: 2 }) 호출
  ↓
3. 응답 PNG base64 → Blob 변환 → 다운로드
  ↓
(병렬)
4. SVG 버전도 함께 생성 (동일 API, format: 'svg')
  ↓
5. 토스트: "PNG와 SVG 2개 파일 저장됨"
```

**의존성**: 이미 구현된 `saveDiagramToFile()` 재활용, 해상도 옵션만 추가
